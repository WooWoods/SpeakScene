# SpeakScene 产品与开发文档

## 1. 产品定位

SpeakScene 是面向中文母语者的 AI 情景英语陪练。核心不是让用户把指定中文句子翻译成英文，而是在一个真实场景中先获得常用表达，再由系统主动发起对话，帮助用户用语音或键入的方式完成一段可持续的英语互动。

目标用户通常知道一些单词，但不知道真实场景里该怎么自然开口。产品要降低“凭空输出完整句子”的压力，让用户可以参考场景短语，在对话中逐步形成表达习惯。

## 2. 核心体验

页面以一个具体场景为中心，桌面端提供高集成的三列工作台，同时支持 PM 视角的交互深度优化：

- **左列（脚手架区）**：场景信息和 LLM 生成的常用表达（中英对照）。每条表达包含英文、中文、使用说明、语气标签。**新增“一键引用”功能**（点击可直接将短语填入输入框），并可一键收藏。
- **中列（对话实践场）**：AI 对话区。系统先发起对话，页面打印英文 starter，并通过 TTS 朗读。用户可通过语音或键入回复。提供“专注模式”（可折叠左右列）和录音状态波形动态反馈，支持在移动端通过左右滑动/标签页切换面板。
- **右列（智能复盘区）**：完成对话后的多维度 AI 评分、最近对话历史、按场景分类的收藏表达，并包含**每日打卡/学习连击（Streak）模块**。

用户不需要先“会写完整句子”才能练习。左侧表达是脚手架，中间对话是实践场，右侧反馈是复盘，三者通过流畅的交互无缝连接。

## 3. MVP 功能范围

- 场景启动：选择等级和类别后，后端生成场景名、中文情境说明、系统开场白和 8-12 条高频表达。
- 对话推进：用户每发送一次回复，后端保存用户轮次并生成下一句系统回复。
- 浏览器语音能力：使用 Web Speech API 做 TTS 和可用浏览器上的 speech recognition；不可用时自动回退到文本输入。
- 语音输入策略：MVP 阶段继续使用浏览器 `SpeechRecognition/webkitSpeechRecognition` 作为低成本语音转文本方案；用户可在提交前查看和编辑识别结果。后续在语音体验需要更稳定、跨浏览器支持更一致时，再接入后端 STT 服务。
- 完成评分：对整段用户对话评分，包括 vocabulary、grammar、authenticity、fluency 和 overall score。
- 收藏系统：用户可收藏真实常用表达，收藏按 scenario category 分组展示。
- 本地开发：默认使用 mock AI client，不需要付费 API key 即可运行完整流程。

## 4. 技术架构

- 前端：React + Vite + Tailwind CSS。主界面是密集型三列工作台，移动端按列堆叠。
- 后端：FastAPI + Pydantic + SQLAlchemy。
- 数据库：SQLite MVP；后续可迁移 PostgreSQL。
- AI 层：当前提供 deterministic mock client；真实 LLM/STT/TTS provider 后续接入 `backend/app/services/ai_client.py`。
- 语音：MVP 使用浏览器 `speechSynthesis` 和 `SpeechRecognition/webkitSpeechRecognition`，不经过后端。
- 后端 STT 规划：后续增加 `POST /api/stt/transcribe`，由前端录制用户音频并提交给后端，后端调用 OpenAI Whisper、ElevenLabs STT、Deepgram 或 AssemblyAI 等 provider，返回 transcript 文本供用户复核、编辑和提交。

## 5. 后端数据模型

MVP 使用以下核心概念：

- ScenarioSession：一次场景练习，包含等级、分类、场景名、中文情境、系统开场白、常用表达、状态、评估结果和完成时间。
- ConversationTurn：一轮对话，包含 speaker、英文文本、可选中文解释、输入方式和时间。
- FavoriteExpression：收藏表达，包含分类、场景名、英文、中文和使用说明。
- User：保留轻量用户字段，认证暂不在本阶段实现。

旧的 PracticeTask / PracticeAttempt / ReviewQueue 模型不再代表当前产品方向。

## 6. API 设计

- `POST /api/scenarios/start`：创建场景会话，返回场景信息、常用表达和系统开场轮次。
- `GET /api/sessions/{session_id}`：获取会话详情和所有轮次。
- `POST /api/sessions/{session_id}/turns`：追加用户回复，返回用户轮次、系统下一轮回复和更新后的会话。
- `POST /api/sessions/{session_id}/complete`：完成对话并返回整段表现评分。
- `GET /api/history`：获取最近场景会话。
- `GET /api/favorites`：按分类获取收藏表达。
- `POST /api/favorites`：收藏表达。
- `DELETE /api/favorites/{favorite_id}`：删除收藏。

## 7. 后续路线

1. **接入真实 LLM**: 替换 mock 场景生成、对话演进和多维度评估，确保对话流畅度与情境真实度。
2. **语音服务升级**:
    - *第一阶段*：保留浏览器 Web Speech API STT/TTS 作为低成本方案。
    - *第二阶段*：接入后端高精度 STT/TTS 服务（如 Whisper, Deepgram, ElevenLabs），保留浏览器原生 STT 作 fallback。
    - *第三阶段*：结合音频原始多模态特征，增加发音评测、语调纠偏、流利度与语速分析。
3. **用户粘性与激励系统 (Stickiness & Retention)**:
    - 引入**每日场景挑战（Daily Scenario Challenge）**与**打卡连击（Practice Streak）**。
    - 上线**智能复习闪卡（SRS Flashcards）**，让收藏表达进入间隔复习算法（如 Leitner 算法）。
    - **上下文唤醒机制 (Active Recall)**：AI 在对话中根据用户已收藏的词组，引导/提示用户在对话中进行主动调用练习。
4. **游戏化成长体系**:
    - 上线**英语交流技能树（Skill Tree）**，包含社交、职场、旅行、应急等分支，通关场景获取技能点与多维度口语雷达图。
5. **账号与云同步**:
    - 增加微信登录、JWT 认证，实现用户练习历史、连击记录与收藏夹跨设备同步。
6. **商业化与自传播**:
    - **练习高光分享卡片 (Share Cards)**：一键生成具有视觉美感的打卡海报，激发微信/小红书等社交媒体分享。
    - Pro 付费权益：无限 AI 场景生成、专属口语教练深度解析、定制高难职场/面试场景包。

## 8. 验收标准

- 首屏直接展示三列核心练习台，无冗余营销干扰。
- 用户进入一个场景后，不需要翻译指定句子，而是参考短语进行真实对话。
- 支持**脚手架“一键引用”**，点击左侧卡片短语可直接填入输入框，大幅降低打字或拼写难度。
- 移动端支持面板流畅切换（手势滑动或 Tab 导航），不得使用繁重的全屏垂直堆叠。
- 支持“专注模式”（快捷键或按钮），一键折叠左右侧边栏，聚焦中间对话。
- 系统开场白与 AI 回复需正常渲染并可自动朗读（支持静音切换）。
- 支持文本输入与语音输入的随时无缝切换；即使后端 STT 服务挂掉，也必须能够平滑回退到浏览器原生识别或键盘输入。
- 录音时有明显的**动态波形动画（Interactive Waveform）**反馈。
- 完成对话后能看到多维度评分和建议表达。
- 收藏表达能按场景分类展示，并可删除。
- 连击（Streak）模块能正确累加打卡天数，并在每次练习完成后有精美的打卡动效。

## 9. PM 视角：UI/UX 与用户粘性（Stickiness）深度优化方案

从产品经理（PM）的角度出发，SpeakScene 的核心价值在于**“消除开口焦虑，建立真实场景中的表达直觉”**。为了让用户不仅“觉得好用”，而且“每天都想来用”，我们需要在 UI/UX 上极致降低认知负载，在机制上建立强大的粘性闭环。

### 9.1 UI/UX 深度优化：降低学习摩擦力（Frictionless Interaction）

#### 1. 脚手架“一键引用”与拼图式表达（Scaffold-to-Input Assembly）
- **痛点**：用户在左侧看到很好用的表达，但要在中间输入框打出来或读出来时，需要频繁转头对照，造成注意力割裂，且极易拼错。
- **优化方案**：
    - 在左侧的每一条常用表达卡片上，增加一个**“引用”按钮（Quote/Assemble）**。
    - 点击后，该短语的英文会高亮飞入（Micro-animation）中间对话框的文本输入区。
    - 用户可以将其作为半成品，在其中修改个别单词（例如更换名词、调整时态），或者直接点击发送。这极大地降低了“从零组织句子”的心理压力。
- **正向反馈**：若用户最终发送的对话中包含/引用了左侧推荐的短语，对话气泡中应点亮**“学以致用 (Expression Match)”**的小勋章，给予用户即时微小成就感。

#### 2. 沉浸式专注模式与移动端触控优化（Focus Mode & Mobile Gestures）
- **痛点**：三列工作台在 13 寸笔记本或平板上可能显得拥挤，信息量过载会加速学习疲劳。
- **优化方案**：
    - **专注模式（Focus Mode）**：在中列顶部设计一个“精简/专注”开关。点击后，左右两侧以侧滑动效折叠收起，中列对话区域横向拉伸，呈现清爽的全屏对话流。
    - **移动端手势化（Mobile Tab-Swipe）**：在移动端弃用单调的长页面滚动。改为**“左-中-右”三屏手势左右滑动**或底部固定导航栏（Phrases | Chat | Stats）。默认停留在 Chat 界面，向右滑查看词卡脚手架，向左滑查看评分与收藏。

#### 3. 闭眼练习模式：Hands-Free Mode
- **痛点**：许多上班族希望在通勤、散步或做家务时闭眼练习口语，此时频繁点击屏幕上的录音和发送键极其繁琐。
- **优化方案**：
    - 引入 **Hands-Free Mode（免手操作模式）**。
    - 利用前端音频流检测（VAD - Voice Activity Detection），当用户说话完毕并停顿超过 1.5 秒时，系统自动切断录音并提交。
    - AI 回复生成后，由高品质 TTS 自动朗读，并自动开启下一轮监听。用户全程无需看屏幕，实现纯语音的“沉浸式对讲机体验”。

### 9.2 用户粘性（Stickiness）增强：建立习惯回路（Habit Loop）

为了提高用户 Retention（留存率），需构建 **“触发（Trigger） -> 行动（Action） -> 可变奖励（Variable Reward） -> 投入（Investment）”** 的习惯回路：

#### 1. 投入（Investment）积累：智能复习闪卡（SRS）与主动召回（Active Recall）
- **核心逻辑**：用户收藏的表达（Investment）绝不应该是“收藏后吃灰”。
- **优化机制**：
    - **智能记忆库（Smart SRS Card Deck）**：右侧收藏栏升级为智能复习卡片。基于间隔重复算法（如 SuperMemo/Leitner），每天为用户推荐 5 个最需要温习的收藏短语。
    - **情境主动召回（Contextual Active Recall）**：这是 SpeakScene 的杀手级粘性特性。当用户启动一个新场景时，后端 AI 引擎获取该用户历史收藏的短语，在生成当前场景的常用表达或对话推进时，**主动引导/提示**用户：“试试用你之前收藏的 `[Expression]` 来回答这一句吧！”这种跨场景的知识串联能让用户体会到极强的个人成长感。

#### 2. 行动与触发：每日精选场景与“成长树”（Daily Challenge & Progression Tree）
- **每日精选（Scenario of the Day）**：每天早上 8:00 推送一个与时事、季节或日常紧密相关的精选场景（例如“春日野餐准备”、“AI 行业面试”），完成挑战可获得“双倍经验值”或限时徽章。
- **技能树路线图（English Mastery Roadmap）**：将所有场景结构化为一张清晰的**职业/生活技能地图**。用户从“生存口语（A1）”起步，逐步解锁“职场进阶（B2）”、“深度社交（C1）”。每一次练习都是在地图上点亮新的节点，提供游戏通关般的爽快感。

#### 3. 可变奖励：多维度成长雷达图与练习海报（Variable Reward & Share Loop）
- **动态口语雷达（Fluency Radar Chart）**：将每次的评分汇总，在个人中心生成**流畅度、语法准确性、地道表达、词汇丰富度**四个维度的雷达图。用户能直观看到随着天数增加，自己的能力雷达边界不断向外扩张。
- **高光复盘分享卡片（Generative Share Card）**：
    - 系统检测到用户打出精妙句子或评分获得 high level 时，在复盘页提供“生成我的英语高光海报”按钮。
    - 一键生成一张设计极简、排版高级、带有一句精彩中英对话及打卡天数的图片，极大满足用户的自我展示（Flexing）心理，促成在微信朋友圈或小红书的自传播。
